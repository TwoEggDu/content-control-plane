# 需要你留意

这个文件不是让你盯每一行代码。

它只回答一件事：

作为项目 owner，当前你最应该留意哪些会影响方向、语义和后续返工成本的东西。

## 怎么使用

你不需要每天完整重读这份文档。

更合适的使用方式是：

- 当我准备进入下一阶段前，看一遍
- 当我提出新决策点时，对照看一遍
- 当你感觉项目开始“做多了”或“做偏了”时，看一遍

## 当前最该留意的 5 类事项

### 1. 第一版范围有没有跑偏

你要盯的是：

- 项目是否还收敛在“质量门禁闭环”
- 我有没有提前把精力拉去做发布控制、权限大系统、复杂报表
- 当前实现是不是还服务于 `M1`

为什么重要：

- 第一版最怕的不是能力少，而是边界失控
- 一旦范围失控，后面的数据模型、API 和实现顺序都会被拖乱

你可以重点看：

- [../roadmap.md](../roadmap.md)
- [../backlog.md](../backlog.md)
- [../product_intent.md](../product_intent.md)

### 2. 导入契约是否稳定

你要盯的是：

- 上游到底能不能稳定提供 `task_no`
- `resource_guid` 和 `resource_path` 哪个更可信
- `branch_name`、`commit_sha`、`scanner_version` 这些字段是否真实可拿到
- 扫描结果里哪些字段只是展示用，哪些字段会进入幂等和指纹

为什么重要：

- 导入契约不稳定，幂等就不稳定
- 幂等不稳定，扫描任务和问题数据就会脏

你可以重点看：

- [../api/01-mvp-api.md](../api/01-mvp-api.md)
- [../api/examples/import-scan-sample.json](../api/examples/import-scan-sample.json)
- [../sdd/02-ingestion-and-issue-domain.md](../sdd/02-ingestion-and-issue-domain.md)

### 3. 问题模型语义对不对

你要盯的是：

- `issue_item` 是否继续作为长期治理对象
- `NEW / ASSIGNED / FIXING / RESOLVED / VERIFIED / IGNORED` 这套状态是否符合你的治理习惯
- `RESOLVED` 和 `VERIFIED` 是否应该分开
- 责任人先用字符串而不是用户外键，你是否接受

为什么重要：

- 这些是后续最容易牵动表结构、API 和工作流的核心语义
- 这类语义如果晚改，返工成本会明显上升

你可以重点看：

- [../data_model.md](../data_model.md)
- [../sdd/03-query-workflow-and-reporting.md](../sdd/03-query-workflow-and-reporting.md)

### 4. 哪些东西现在明确不该做

你要盯的是：

- 我有没有提前进入发布控制主线
- 我有没有过早做复杂统计和大盘
- 我有没有开始做重权限体系
- 我有没有为了“看起来完整”而引入不必要的基础设施

为什么重要：

- 当前阶段真正值钱的是把 `M1` 做成可靠基线
- 不是先把平台外壳铺满

如果你发现实现开始碰这些内容，就应该直接提醒我收回来。

### 5. 不可逆或高返工成本的基础决策

你要盯的是：

- 表结构和主关系是否合理
- migration 是否会让后续扩展很难做
- PostgreSQL 和对象存储的职责边界是否清楚
- 运行约束是否符合你的实际使用方式

为什么重要：

- 这部分是从样例工程走向真实后端的分水岭
- 一旦这里拍错，后面修成本会比文档阶段高很多

你可以重点看：

- [../sdd/05-runtime-storage-and-ops.md](../sdd/05-runtime-storage-and-ops.md)
- [../data_model.md](../data_model.md)
- [../runbook/README.md](../runbook/README.md)

## 当前你不用太盯的东西

这些我可以默认自己推进，你不需要频繁介入：

- `Gin` 路由怎么拆
- 内部目录怎么微调
- 内存仓储如何换成 PostgreSQL 仓储
- 一般性的测试补充
- 常规重构和命名整理

## 什么时候你应该介入

出现这些情况时，你最好及时看一下：

- 我提出新的 owner 决策项
- 我说要调整数据模型核心语义
- 我说要改变第一版范围
- 我说要引入新的外部基础设施
- 我说要开始做发布控制或权限系统

## 当前阶段一句话提醒

你现在最该盯的是：

- 范围是否收敛
- 导入契约是否真实
- 问题模型语义是否对
- 基础持久化决策是否稳

而不是具体某个 Handler 或某个小重构写得漂不漂亮。
