# Runbook

## 文档目的

这个目录用于存放运行和操作手册。

Runbook 回答的不是“系统怎么设计”，而是：

- 这个系统怎么启动
- 常见操作怎么执行
- 出问题时先查什么
- 哪些步骤需要标准化

## 当前阶段

当前仓库还没有进入正式运行阶段，因此这里先建立入口和使用原则。

## 后续建议补充的 Runbook

### 1. 本地启动手册

适用时机：Go 项目骨架落地后

内容应包括：

- 依赖准备
- 数据库启动
- 配置项说明
- 服务启动方式

### 2. 导入验证手册

适用时机：导入接口落地后

内容应包括：

- 如何准备样例 JSON
- 如何执行导入
- 如何检查导入结果
- 常见失败原因

### 3. 迁移执行手册

适用时机：数据库迁移引入后

内容应包括：

- 如何执行 migration
- 如何回滚 migration
- 如何在本地清空并重建环境

### 4. 对象存储检查手册

适用时机：附件和报告接入后

内容应包括：

- 上传失败怎么排查
- 存储 key 如何定位
- 如何验证元信息和实际对象一致

### 5. 发布和回滚手册

适用时机：发布控制模块落地后

内容应包括：

- 发布前检查项
- 回滚条件
- 异常后第一响应动作

## 维护规则

- 重复超过两次的手工操作，应考虑沉淀为 Runbook
- 出现事故或排障过程后，应优先补 Runbook
- Runbook 应偏操作步骤，不应写成大段设计说明
